# povo Manager

povo 2.0 の複数回線をまとめて管理する Android アプリです。  
最終トッピング購入日から 180 日の自動解約日を自動計算し、期限が近づいたらプッシュ通知でお知らせします。

## 主な機能

### 回線・期限管理
- povo 回線を複数登録して一覧管理
- 最終トッピング購入日から自動解約日(デフォルト 180 日)を自動計算
- ホーム画面の各回線カードに残日数バッジと進捗バーを表示
- 表示名・メモを自由に設定可能

### トッピング購入記録
- 購入日・トッピング名・有効期限を記録
- 主要トッピングのプリセット選択に対応(自由入力も可能)
- 有効期間中の最も期限が近いトッピングを回線カードに表示
- 履歴の修正・削除および Snackbar からの操作取り消し

**プリセット一覧**

| トッピング | 有効日数 |
|---|---|
| データ追加1GB(7日間) | 7日 |
| データ追加3GB(30日間) | 30日 |
| データ追加20GB(30日間) | 30日 |
| データ追加60GB(90日間) | 90日 |
| データ追加150GB(180日間) | 180日 |
| データ使い放題(24時間) | 1日 |
| 5分以内通話かけ放題 | 期限なし |
| 通話かけ放題 | 期限なし |

### 通知
- 自動解約日の X 日前に通知(共通設定: 30・14・7・3・1・0 日前)
- 回線ごとに通知タイミングを個別設定で上書き可能
- トッピング有効期限の X 日前にも通知(デフォルト: 3・1 日前)
- 通知時刻は 0〜23 時から選択
- 期限超過時は毎日警告を継続

### ホームスクリーンウィジェット
- 期限が最も近い回線を大きく表示し、残り最大 4 回線をコンパクトに表示
- 残日数に応じて文字色が変化(7 日以内は赤)
- タップするとアプリが開く

### カレンダー登録
- 回線詳細画面から自動解約日を Android カレンダーアプリへ一発登録

### バックアップ / リストア
- 全回線と購入履歴を JSON ファイルにエクスポート
- インポートは 2 つのモードを選択可能
  - **上書き**: 同じ電話番号の回線を更新し、新しい回線を追加(既存データを保持)
  - **全置換**: 現在のデータをすべて削除してファイルの内容に置き換え

### 設定
| 項目 | デフォルト | 変更範囲 |
|---|---|---|
| 自動解約日 通知タイミング | 30・14・7・3・1・0 日前 | 60・30・14・7・3・1・0 日前から選択 |
| トッピング期限 通知タイミング | 3・1 日前 | 7・3・1・0 日前から選択 |
| 通知時刻 | 9:00 | 0:00〜23:00 |
| 解約までの日数 | 180 日 | 1〜3650 日(povo 規約変更時に調整用) |

## 技術スタック

- **言語**: Kotlin
- **UI**: Jetpack Compose + Material 3
- **DB**: Room
- **DI**: Hilt
- **設定永続化**: DataStore
- **バックグラウンド通知**: WorkManager
- **ウィジェット**: Glance

## ビルド / リリース

GitHub Actions により、GitHub リリースを公開すると署名済み APK が自動でビルドされ、リリースアセットとして添付されます。

ビルドに必要な Secrets:

| Secret | 内容 |
|---|---|
| `KEYSTORE_BASE64` | キーストアファイルの Base64 エンコード |
| `KEYSTORE_PASSWORD` | キーストアのパスワード |
| `KEY_ALIAS` | 署名キーのエイリアス |
| `KEY_PASSWORD` | 署名キーのパスワード |
